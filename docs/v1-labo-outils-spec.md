# V1 Outils Du Labo

## Intention

Définir les outils de base du labo en V1.

Le labo ne sert pas à afficher du contenu abstrait.
Il sert à donner à l'utilisateur des objets concrets, activables et personnels pour mieux traverser sa transformation.

La V1 retenue repose sur 2 grandes familles d'outils :

- `les cartes`
- `les potions d'état`

Les cartes retenues sont :

- `carte de défense`
- `carte d'attaque`
- `carte de soutien`

Les potions retenues sont :

- `potion de rappel`
- `potion de courage`
- `potion de guérison`
- `potion de clarté`
- `potion d'amour`
- `potion d'apaisement`

Pas plus en V1.

L'objectif est d'avoir un système :

- simple à comprendre
- rapide à activer
- facile à personnaliser
- suffisamment souple pour couvrir des cas très différents
- compatible avec le plan comme orchestrateur dynamique

## Place Du Labo Dans Le Produit

Le labo fait partie des `supports`.

Cela signifie que le plan peut envoyer l'utilisateur dans le labo pour :

- créer une carte
- valider une carte préremplie
- corriger une carte existante
- activer un support au bon moment

Le labo n'est pas un fourre-tout d'outils complexes.
En V1, il doit rester centré sur quelques objets très clairs.

## Les Deux Familles D'Outils

## 1. Les Cartes

Les cartes sont :

- stables
- personnalisées
- préparées à l'avance
- conçues pour être réutilisées

Une carte est un objet structuré que l'utilisateur peut retrouver quand il en a besoin.

## 2. Les Potions D'État

Les potions sont :

- ponctuelles
- activées à chaud
- liées à l'état du moment
- conçues pour apporter un effet rapide

Une potion n'est pas une carte bis.
Une potion est une réponse courte à un besoin immédiat.

La distinction de base est :

- `carte` = plan préparé à l'avance
- `potion` = réponse immédiate à un état présent

## Les 3 Cartes Du Labo

## 1. Carte De Défense

### Définition

La carte de défense sert quand un contexte à risque arrive.

Question centrale :

- "quand cette situation arrive, comment je me protège sans partir en vrille ?"

Elle est :

- réactive
- protectrice
- anti-bascule
- utile dans les moments où le danger devient concret

### Bonne Granularité

Règle fondamentale :

- `1 carte de défense = 1 situation ou 1 contexte précis`

Pas :

- une carte pour tous les problèmes
- une carte générique "quand ça va mal"
- une carte avec 5 scénarios différents mélangés

Exemples corrects :

- quand je suis seul le soir
- quand je sors du travail stressé
- quand je pense à écrire à cette personne
- quand je vais me coucher et que mon cerveau s'emballe
- quand j'ai envie de fumer après le repas

Si plusieurs contextes sont importants :

- plusieurs cartes

### Rôle Produit

La carte de défense doit aider l'utilisateur à :

- reconnaître le scénario avant qu'il déborde
- identifier les signes précoces
- savoir quoi faire immédiatement
- éviter la panique ou la confusion

### Les 3 Blocs Retenus

La structure V1 conserve 3 blocs :

- `stratège`
- `surveillant`
- `défenseur`

### Bloc `stratège`

Rôle :

- comprendre le scénario
- nommer le contexte
- rendre le danger lisible

Questions utiles :

- dans quel contexte précis cette carte s'active ?
- qu'est-ce qui se joue vraiment ici ?
- pourquoi ce moment est dangereux pour moi ?

Ce bloc ne doit pas devenir une dissertation.
Il doit donner une lecture courte et nette du scénario.

### Bloc `surveillant`

Rôle :

- repérer les signes précoces
- détecter la montée avant la chute

Questions utiles :

- comment je sais que ça commence ?
- quelles pensées reviennent ?
- quels gestes, justifications ou sensations annoncent la bascule ?

Exemples :

- "je commence à me dire que ce n'est pas si grave"
- "je tourne en rond"
- "je sens une pression dans la poitrine"
- "je cherche déjà une excuse"

### Bloc `défenseur`

Rôle :

- donner la réponse immédiate
- transformer l'analyse en geste concret

Questions utiles :

- qu'est-ce que je fais maintenant ?
- quel est le bon réflexe à la place du mauvais ?
- quel est le plus petit geste qui me protège réellement ?

Exemples :

- sortir marcher 5 minutes
- envoyer un message à une personne ressource
- quitter la pièce
- poser le téléphone
- boire un verre d'eau et relire sa phrase de rappel

### Ce Que La Carte Ne Doit Pas Être

- un texte trop long
- une fiche trop intellectuelle
- une analyse psychologique générale
- une check-list de 12 étapes

La carte doit rester :

- lisible en quelques secondes
- activable immédiatement
- personnelle

## 2. Carte D'Attaque

### Définition

La carte d'attaque sert à prendre de l'avance.

Question centrale :

- "comment je crée les bonnes dispositions avant que ça déraille ?"

Elle est :

- proactive
- préparatoire
- orientée élan
- orientée conditionnement

### Rôle Produit

La carte d'attaque doit aider l'utilisateur à :

- se mettre dans le bon état
- préparer le terrain
- faire travailler son attention et son inconscient dans le bon sens
- rendre le bon comportement plus probable

### Ce Que La Carte D'Attaque N'Est Pas

- une réaction d'urgence
- un plan de crise
- un simple mantra décoratif

Elle doit produire un vrai avantage avant le moment difficile.

### Types D'Actions Pertinents

Exemples cohérents :

- un texte court à réciter 3 fois le matin
- une phrase très courte répétée matin, midi et soir
- un texte à écrire jusqu'à ce que la compulsion ou l'envie baisse
- un geste symbolique de contrat avec soi-même
- un mini-rituel avant une situation sensible
- une séquence de préparation avant une conversation difficile

### Structure Recommandée

La carte d'attaque peut être structurée ainsi :

- `moment cible`
- `intention`
- `déclencheur`
- `action d'attaque`

### Bloc `moment cible`

Rôle :

- préciser quand l'attaque se joue

Exemples :

- le matin
- avant de sortir
- avant une soirée à risque
- avant de se coucher
- avant un entretien

### Bloc `intention`

Rôle :

- nommer ce qu'on cherche à installer

Exemples :

- retrouver de la fermeté
- entrer dans la journée avec une direction claire
- ne pas laisser la fatigue décider
- me rappeler qui je veux être

### Bloc `déclencheur`

Rôle :

- définir le moment d'activation

Exemples :

- après le réveil
- avant de quitter la maison
- dès que je sens une hésitation
- juste avant le contexte à risque

### Bloc `action d'attaque`

Rôle :

- donner le geste actif qui met de l'avance

Exemples :

- réciter un texte court
- écrire une phrase plusieurs fois
- faire un point sur le poignet comme engagement
- relire un engagement personnel
- lancer un mini-rituel d'entrée

### Règle De Design

Une bonne carte d'attaque doit :

- être simple
- être assumable
- créer une sensation d'intention
- être répétable sans rejet

Elle ne doit pas :

- sembler ridicule ou infantilisante
- demander trop d'énergie
- ressembler à une corvée

## 3. Carte De Soutien

### Définition

La carte de soutien sert quand l'état intérieur se dégrade.

Question centrale :

- "quand je m'effondre un peu à l'intérieur, comment je me soutiens au lieu de me laisser couler ?"

Elle est :

- émotionnelle
- contenante
- régulatrice
- utile dans les moments de creux

### Quand Elle Est Particulièrement Pertinente

La carte de soutien est très utile quand la transformation contient :

- tristesse
- démotivation
- honte
- vide
- fatigue morale
- envie de tout lâcher
- impression d'échec
- découragement diffus

Elle est particulièrement adaptée pour :

- deuil
- anxiété
- confiance / estime
- solitude
- transitions de vie
- fatigue émotionnelle

### Différence Avec Les Deux Autres Cartes

La distinction doit rester nette :

- `défense` = je me protège d'un scénario à risque
- `attaque` = je prends de l'avance
- `soutien` = je me tiens quand intérieurement je chute

### Rôle Produit

La carte de soutien doit aider l'utilisateur à :

- reconnaître son état émotionnel réel
- éviter les décisions prises dans le creux
- retrouver un minimum d'appui
- revenir à un tout petit pas possible

### Structure Recommandée

La carte de soutien peut être structurée ainsi :

- `état`
- `vérité à me rappeler`
- `geste de soutien`
- `interdit`
- `point de reprise`

### Bloc `état`

Rôle :

- nommer comment je sais que je suis dedans

Exemples :

- "quand je me sens nul"
- "quand j'ai envie de tout lâcher"
- "quand je me sens vide et sans force"
- "quand je me dis que ça ne sert à rien"

### Bloc `vérité à me rappeler`

Rôle :

- apporter une phrase juste dans le creux

Exemples :

- "un creux ne veut pas dire que j'ai régressé"
- "je n'ai pas besoin d'être fort aujourd'hui, juste de ne pas me laisser tomber"
- "ce soir n'est pas toute mon histoire"

### Bloc `geste de soutien`

Rôle :

- donner l'acte le plus simple et le plus aidant

Exemples :

- marcher 5 minutes
- écouter un audio calmant
- relire son pourquoi
- boire quelque chose et s'asseoir au calme
- envoyer un message à une personne sûre

### Bloc `interdit`

Rôle :

- empêcher les faux gestes typiques de cet état

Exemples :

- ne pas prendre de décision radicale ce soir
- ne pas écrire à cette personne
- ne pas me juger à partir d'un moment de fatigue

### Bloc `point de reprise`

Rôle :

- redonner une sortie par le bas
- offrir un retour minuscule mais réel vers la trajectoire

Exemples :

- faire une seule action minuscule
- lancer le plus petit rituel possible
- revenir à une action de 2 minutes

## Ce Qu'On Ne Garde Pas Comme Carte De Base En V1

## Pas De Carte `comptable` Centrale

Le `comptable` ne doit pas structurer le labo en V1.

Raisons :

- trop de friction cognitive
- valeur inégale selon les profils
- complexité produit inutile à ce stade

Si on conserve une trace d'usage :

- elle doit rester minimale
- par exemple un simple bouton `+1 je l'ai utilisée`

Mais ce n'est pas un bloc narratif central de la carte.

## Pas De Multiplication Des Familles De Cartes

En V1, il faut éviter :

- carte de reprise
- carte de clarté
- carte de préparation
- sous-familles multiples

Elles risqueraient de rendre le labo confus.

Le bon noyau V1 est :

- `défense`
- `attaque`
- `soutien`

## Les Potions D'État

## Définition Générale

Les potions sont des processus courts qui s'activent en fonction de l'état ou de l'émotion de la personne sur le moment.

Leur rôle n'est pas de remplacer :

- les cartes
- le plan
- les habitudes

Leur rôle est de :

- réguler
- réorienter
- remettre debout
- redonner une ressource immédiate
- réactiver une meilleure disposition

## Logique Produit

Quand un utilisateur clique sur une potion, le flow recommandé est :

1. mini-questionnaire
2. champ libre
3. réponse immédiate de Sophia
4. décision éventuelle de suivi sur les jours suivants

La potion doit donc produire deux choses :

- un effet immédiat
- un prolongement éventuel si le besoin le justifie

## Mini-Questionnaire Recommandé

Le mini-questionnaire doit rester très léger.

Il doit servir à comprendre :

- ce qui se passe maintenant
- depuis quand
- ce dont la personne a besoin

Format recommandé :

- 1 à 3 questions courtes
- 1 champ libre optionnel ou semi-obligatoire selon la potion

Exemples de questions :

- qu'est-ce qui se passe pour toi maintenant ?
- depuis quand tu te sens comme ça ?
- de quoi tu aurais le plus besoin là tout de suite ?
- est-ce que c'est plutôt un creux, une peur, un brouillard, une solitude ou un emballement ?

## Réponse Immédiate De Sophia

Après le mini-questionnaire, Sophia doit générer un petit pavé très ciblé.

Ce pavé doit :

- répondre au besoin du moment
- apporter du soulagement ou de l'orientation
- être utile immédiatement
- rester court

Ce n'est pas une dissertation.
Ce n'est pas une leçon.
C'est un geste d'aide instantané.

## Suivi Éventuel

Après la réponse immédiate, le système peut décider :

- de ne rien ajouter
- de créer un `rendez-vous` unique
- de créer une petite série de messages sur quelques jours

Exemples :

- potion d'amour -> un message de soutien chaque jour pendant 3 à 7 jours
- potion de guérison -> un rappel doux le lendemain
- potion de clarté -> un point bref dans 48h pour vérifier si la route est plus nette

Le suivi ne doit pas être systématique.
Il doit être déclenché quand il y a une vraie valeur.

## Lien Avec Le Système Existant

Le code contient déjà deux rails utiles :

- `rendez-vous`
- `scheduled_checkins`

Donc la bonne logique produit n'est pas de créer un système parallèle pour les potions.

La potion doit :

- créer l'aide immédiate
- puis, si besoin, s'appuyer sur `rendez-vous` et/ou `scheduled_checkins`

## Les 6 Potions V1

## 1. Potion De Rappel

### État Déclencheur

- la personne décroche de son cap
- elle oublie pourquoi elle fait tout ça
- elle sent que ça sort de sa tête
- elle perd le fil de sa transformation

### Effet Recherché

- remettre du cap
- rappeler le sens
- reconnecter à la trajectoire

### Ce Que La Potion Doit Produire

- un rappel du pourquoi
- une reformulation de ce qu'elle est en train de faire
- un retour au prochain petit pas juste

## 2. Potion De Courage

### État Déclencheur

- la personne n'ose pas
- elle évite
- elle repousse
- elle manque de fermeté
- elle sent qu'elle va se dégonfler

### Effet Recherché

- remettre de l'élan
- restaurer la capacité à passer à l'acte
- sortir de l'évitement

### Ce Que La Potion Doit Produire

- une remise en tension juste
- une parole qui redonne du nerf
- un prochain geste concret

## 3. Potion De Guérison

### État Déclencheur

- la personne a trébuché
- elle a merdé
- elle n'a pas tenu
- elle culpabilise
- elle pense que tout est foutu

### Effet Recherché

- réparer
- dégonfler la honte
- empêcher l'abandon global
- remettre dans une logique de continuité

### Ce Que La Potion Doit Produire

- une lecture non catastrophique
- une parole de réparation
- un point de reprise minuscule

## 4. Potion De Clarté

### État Déclencheur

- la personne est dans le brouillard
- elle ne sait plus quoi faire
- elle hésite entre plusieurs directions
- elle se perd dans sa tête

### Effet Recherché

- réduire le brouillard
- remettre une priorité
- retrouver une direction simple

### Ce Que La Potion Doit Produire

- une simplification
- une prochaine direction
- une action suivante évidente

## 5. Potion D'Amour

### État Déclencheur

- la personne se sent seule
- elle se sent abandonnée
- elle se sent peu aimée
- elle a besoin de chaleur humaine
- elle manque de soutien affectif

### Effet Recherché

- redonner de la chaleur
- contenir la solitude
- remettre du lien
- offrir un appui émotionnel doux

### Ce Que La Potion Doit Produire

- une présence verbale forte
- une sensation d'être rejointe
- un geste concret de douceur ou de reliance

## 6. Potion D'Apaisement

### État Déclencheur

- anxiété
- emballement mental
- agitation
- tension nerveuse
- impression d'être submergé

### Effet Recherché

- redescendre
- calmer le système
- recréer un peu d'espace intérieur

### Ce Que La Potion Doit Produire

- une décélération
- une parole calmante
- un geste de régulation très simple

## Structure Produit Recommandée Pour Une Potion

Chaque potion doit avoir :

- `state_trigger`
- `mini_questionnaire`
- `free_text`
- `instant_response`
- `follow_up_strategy`

## `state_trigger`

L'état reconnu par l'utilisateur ou détecté dans l'interface.

## `mini_questionnaire`

Les 1 à 3 questions courtes de cadrage.

## `free_text`

L'espace où l'utilisateur dit ce qui se passe dans ses mots.

## `instant_response`

Le petit pavé de Sophia.

## `follow_up_strategy`

La décision de prolongement :

- aucun suivi
- un `rendez-vous`
- une petite séquence de messages

## Règle De Design

Une bonne potion doit être :

- rapide
- nette
- ciblée
- utile immédiatement

Elle ne doit pas :

- ressembler à une carte
- demander trop d'effort
- se transformer en tunnel long
- noyer la personne dans l'analyse

## Différence Nette Cartes Vs Potions

### Carte de défense

- me protège dans un contexte précis

### Carte d'attaque

- me met en avance avant un contexte

### Carte de soutien

- me soutient dans un creux intérieur

### Potion

- m'aide à traverser un état immédiat ici et maintenant

## Règles Transversales De Construction

Quel que soit le type de carte :

- la carte doit être courte
- la carte doit être personnelle
- la carte doit être activable rapidement
- la carte doit conduire à un geste concret
- la carte doit être compréhensible sur mobile

## Préremplissage

Le système peut préparer une première version à partir :

- du questionnaire
- de la transformation active
- des patterns de difficulté
- des messages récurrents
- de la classification de type de plan

Mais la règle produit est :

- `préparer si la confiance est suffisante`
- `faire valider ou corriger par l'utilisateur`

Il ne faut pas imposer silencieusement une carte.

## Validation Utilisateur

Le labo doit tirer parti de la logique suivante :

- Sophia prépare
- l'utilisateur valide ou ajuste
- cette validation compte comme une vraie avancée

Cela est particulièrement important en phase 1, où l'utilisateur doit sentir un progrès rapide et concret.

## Frontière Avec L'Architecte

Le labo n'a pas vocation à porter tous les outils introspectifs profonds.

Le labo est fait pour :

- protéger
- préparer
- soutenir
- activer

Des outils plus profonds de restructuration intérieure ou de réparation devraient plutôt vivre côté `Architecte`.

Exemples plus proches de l'Architecte :

- relecture d'un souvenir traumatique
- travail de pardon structuré
- outils longs de reconfiguration intérieure
- exercices qui demandent une élaboration importante

Le critère simple est :

- si l'objet doit être activé vite dans le quotidien -> plutôt `labo`
- si l'objet demande un travail plus profond, plus long, plus délicat -> plutôt `Architecte`

## Lien Avec Le Plan

Dans le plan, les cartes du labo appartiennent à la catégorie :

- `supports`

Le plan peut donc contenir des actions comme :

- aller valider une carte de défense
- compléter une carte d'attaque
- revoir une carte de soutien préremplie

Ces items peuvent être :

- `open_module`
- `review_prefilled_tool`
- `complete_tool`

Les potions peuvent aussi être déclenchées depuis le plan comme `supports`, mais leur logique naturelle est souvent plus contextuelle et plus spontanée que celle des cartes.

## Place Dans La Phase 1 Universelle

Le socle universel recommandé après validation du plan est :

- 1 carte de défense
- 1 carte d'attaque
- passage par `Inspiration`

La carte de soutien n'a pas besoin d'être systématiquement obligatoire en phase 1.
Elle doit être fortement recommandée quand la transformation ou les patterns montrent une vraie vulnérabilité émotionnelle.

## Règle V1 Simple

Au lancement :

- tout le monde doit pouvoir construire ou valider une carte de défense
- tout le monde doit pouvoir construire ou valider une carte d'attaque
- certaines personnes doivent aussi recevoir une carte de soutien pertinente
- tout le monde n'a pas besoin d'utiliser une potion immédiatement
- mais les 6 potions d'état doivent être disponibles dans le labo comme couche d'aide rapide

## Sortie Attendue Pour L'Utilisateur

Quand le labo V1 est bien conçu, l'utilisateur doit en sortir avec :

- un réflexe de protection
- un réflexe de préparation
- et, si nécessaire, un réflexe de soutien émotionnel

Autrement dit :

- quoi faire quand ça monte
- quoi faire pour prendre de l'avance
- quoi faire quand intérieurement ça baisse

Et en plus, le labo doit lui permettre de trouver rapidement :

- quoi faire quand il perd le fil
- quoi faire quand il manque de courage
- quoi faire quand il culpabilise
- quoi faire quand il est dans le brouillard
- quoi faire quand il se sent seul
- quoi faire quand il est submergé
